# Shop App - Контрольная работа №2

## Описание проекта

Полнофункциональное веб-приложение для управления товарами с системой аутентификации, авторизации на основе ролей (RBAC) и JWT-токенами.

### Реализованный функционал

#### Backend (Node.js + Express)

1. **Аутентификация**
   - Регистрация пользователей с хешированием паролей (bcrypt)
   - Вход в систему с выдачей JWT-токенов
   - Автоматическое обновление access- и refresh-токенов
   - Защита маршрутов с помощью middleware

2. **Система ролей (RBAC)**
   - **Гость** - неаутентифицированный пользователь (регистрация, вход, обновление токенов)
   - **Пользователь** - просмотр товаров, получение информации о себе
   - **Продавец** - управление товарами (создание, редактирование)
   - **Администратор** - полное управление товарами и пользователями

3. **API маршруты**

| Маршрут | Метод | Доступ | Описание |
|---------|-------|--------|----------|
| `/api/auth/register` | POST | Гость | Регистрация пользователя |
| `/api/auth/login` | POST | Гость | Вход в систему |
| `/api/auth/refresh` | POST | Гость | Обновление пары токенов |
| `/api/auth/me` | GET | Пользователь | Информация о текущем пользователе |
| `/api/users` | GET | Администратор | Список пользователей |
| `/api/users/:id` | GET | Администратор | Пользователь по ID |
| `/api/users/:id` | PUT | Администратор | Обновить пользователя |
| `/api/users/:id` | DELETE | Администратор | Заблокировать пользователя |
| `/api/products` | GET | Пользователь | Список товаров |
| `/api/products/:id` | GET | Пользователь | Товар по ID |
| `/api/products` | POST | Продавец | Создать товар |
| `/api/products/:id` | PUT | Продавец | Обновить товар |
| `/api/products/:id` | DELETE | Администратор | Удалить товар |

#### Frontend (React)

1. **Страницы аутентификации**
   - Вход в систему
   - Регистрация нового пользователя

2. **Управление товарами**
   - Просмотр списка товаров (карточки)
   - Создание нового товара (продавец/админ)
   - Редактирование товара (продавец/админ)
   - Удаление товара (админ)

3. **Управление пользователями (админ)**
   - Просмотр списка пользователей (таблица)
   - Изменение роли пользователя
   - Редактирование данных пользователя
   - Блокировка пользователя

4. **Автоматическая работа с токенами**
   - Автоматическая подстановка access-токена в заголовки запросов
   - Автоматическое обновление токенов при получении 401 ошибки
   - Перенаправление на страницу входа при истечении refresh-токена

## Технологии

### Backend
- **Node.js** - среда выполнения
- **Express** - веб-фреймворк
- **bcrypt** - хеширование паролей
- **jsonwebtoken** - работа с JWT
- **cors** - поддержка CORS
- **nanoid** - генерация уникальных ID

### Frontend
- **React 18** - UI-библиотека
- **React Router DOM** - маршрутизация
- **Axios** - HTTP-клиент с перехватчиками
- **Context API** - управление состоянием аутентификации

## Как запустить

### Требования
- Node.js (версия 16 или выше)
- npm

> **Если Node.js не установлен:**
> 1. Скачайте установщик с официального сайта: https://nodejs.org/
> 2. Рекомендуется скачать LTS-версию (Long Term Support)
> 3. Запустите установщик и следуйте инструкциям
> 4. После установки перезапустите терминал
> 5. Проверьте установку командами: `node -v` и `npm -v`

### Запуск Backend

1. Откройте терминал и перейдите в папку `backend`:
```bash
cd backend
```

2. Установите зависимости:
```bash
npm install
```

3. Запустите сервер:
```bash
npm start
```

Backend будет доступен по адресу: `http://localhost:3000`

### Запуск Frontend

1. Откройте новый терминал и перейдите в папку `frontend`:
```bash
cd frontend
```

2. Установите зависимости:
```bash
npm install
```

3. Запустите приложение:
```bash
npm run dev
```

Frontend будет доступен по адресу: `http://localhost:3001`

> **Примечание:** Если `npm` не распознаётся как команда, добавьте Node.js в PATH или используйте полный путь:
> ```bash
> set PATH=C:\Program Files\nodejs;%PATH%
> ```

## Как пользоваться

1. **Регистрация**: Перейдите на страницу регистрации и создайте аккаунт. По умолчанию новый пользователь получает роль "user" (Пользователь).

2. **Вход**: Войдите с использованием email и пароля.

3. **Управление товарами**:
   - Все аутентифицированные пользователи могут просматривать товары
   - Продавцы и администраторы могут создавать и редактировать товары
   - Только администраторы могут удалять товары

4. **Управление пользователями** (только для администраторов):
   - Просмотр списка пользователей
   - Изменение ролей (user/seller/admin)
   - Редактирование данных пользователей
   - Блокировка пользователей

## Тестовые аккаунты

Для тестирования можно создать пользователей с разными ролями:

1. Зарегистрируйте обычного пользователя (роль `user` по умолчанию)
2. Через базу данных (массив `users` в `server.js`) или через API с ролью `admin` создайте администратора
3. Создайте продавца с ролью `seller`

Пример запроса на регистрацию администратора:
```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@test.com",
    "first_name": "Админ",
    "last_name": "Системы",
    "password": "admin123",
    "role": "admin"
  }'
```

## Архитектура работы

### Процесс аутентификации

```
1. Клиент → POST /api/auth/login → Сервер
2. Сервер проверяет email и пароль (bcrypt.compare)
3. Сервер генерирует:
   - access_token (живёт 15 минут)
   - refresh_token (живёт 7 дней)
4. Клиент сохраняет оба токена в localStorage
5. При каждом запросе access_token добавляется в заголовок:
   Authorization: Bearer {access_token}
6. При 401 ошибке:
   - Клиент отправляет refresh_token на POST /api/auth/refresh
   - Сервер проверяет refresh_token и выдаёт новую пару токенов
   - Клиент повторяет исходный запрос с новым access_token
```

### Система ролей

```
Гость (не аутентифицирован)
  └─ Регистрация, вход, обновление токенов

Пользователь (user)
  └─ Просмотр товаров + всё от Гостя
  └─ /api/auth/me

Продавец (seller)
  └─ Создание и редактирование товаров + всё от Пользователя

Администратор (admin)
  └─ Управление пользователями, удаление товаров + всё от Продавца
```
